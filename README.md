# k-system
```sh
.
├── knowledge/          # Base de conhecimento em Markdown (índices, áreas, ideias)
├── references/         # Referências acadêmicas (Zotero, BibTeX, anotações)
├── datasets/           # Dados brutos, processados, derivados e artefatos
├── database/           # Esquemas SQL, migrações e dumps do PostgreSQL
├── orchestration/      # Airflow DAGs e scripts de automação
├── generation/         # Prompts e outputs de LLMs (texto, imagens, templates)
├── integration/        # Código de integração entre módulos (Zotero, Obsidian, etc.)
├── experiments/        # Notebooks, protótipos e ideias descartadas
├── meta/               # Documentação do projeto (arquitetura, decisões, convenções)
├── infra/              # Configuração Docker e variáveis de ambiente
└── docker-compose.yml  # Orquestração dos containers (PostgreSQL + Airflow)
```
